# Task: Fix Swipe Double-Trigger on Mobile

## Objective
Fix the issue where swiping on mobile calendar views changes months/weeks/days twice.

## Current State
- `useSwipeNavigation` hook handles swipe detection
- Used in: CalendarGrid (Month), WeekView, DayView
- Issues identified:
  1. No debouncing/cooldown between swipe triggers
  2. Complex mobile logic in WeekView may double-call
  3. Touch events may fire multiple times on some devices

## Root Causes

### Cause 1: No Cooldown in Hook (useSwipeNavigation.ts)
```typescript
const handleTouchEnd = useCallback((e: React.TouchEvent) => {
  if (touchStartX.current === null) return
  
  const touchEndX = e.changedTouches[0].clientX
  const diff = touchStartX.current - touchEndX
  
  if (Math.abs(diff) > 50) {
    const direction = diff > 0 ? 'next' : 'prev'
    onNavigate(direction)  // Called immediately, no cooldown
  }
  touchStartX.current = null
}, [onNavigate])
```

### Cause 2: WeekView Complex Logic (WeekView.tsx lines 105-163)
```typescript
const handleMobileTouchEnd = useCallback(
  (e: React.TouchEvent) => {
    if (!isMobile) {
      swipeTouchEnd(e)  // Trigger 1

    }

    if ((diff > 0 && isAtEnd) || (diff < 0 && isAtStart)) {
```

### Cause 3: Touch Event Bubbling
Touch events may bubble up and trigger multiple handlers.

## Requirements
1. Add cooldown/debounce to prevent rapid re-triggering
2. Ensure single swipe = single navigation
3. Maintain responsive feel (no lag)
4. Work consistently across all views

### Solution: Add Cooldown to Hook

```typescript
// useSwipeNavigation.ts
export function useSwipeNavigation(options: SwipeOptions) {
  const { onNavigate } = options
  const touchStartX = useRef<number | null>(null)
  const lastSwipeTime = useRef<number>(0)
  const SWIPE_COOLDOWN = 500  // ms
  
  const handleTouchStart = useCallback((e: React.TouchEvent) => {
    touchStartX.current = e.touches[0].clientX
  }, [])
  
  const handleTouchEnd = useCallback((e: React.TouchEvent) => {
    if (touchStartX.current === null) return
    
    // Check cooldown
    const now = Date.now()
    if (now - lastSwipeTime.current < SWIPE_COOLDOWN) {
      touchStartX.current = null
      return
    }
    
    const touchEndX = e.changedTouches[0].clientX
    const diff = touchStartX.current - touchEndX
    const threshold = 50
    
    if (Math.abs(diff) > threshold) {
      const direction = diff > 0 ? 'next' : 'prev'
      lastSwipeTime.current = now
      onNavigate(direction)
    }
    
    touchStartX.current = null
  }, [onNavigate])
  
  return { handleTouchStart, handleTouchEnd }
}
```

Simplify the WeekView touch end handler to remove potential double-call:

```typescript

const handleTouchEnd = useCallback(
  (e: React.TouchEvent) => {
```

Remove the complex mobile scroll-position checking that's causing issues, OR ensure it only calls swipe handler once.

### Alternative: CSS touch-action
Prevent browser handling of touch gestures:
