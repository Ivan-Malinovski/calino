# Task: Add RELATIVE-TO Property to EventModal

## Objective
Add support for the VEVENT RELATIVE-TO property in the EventModal, allowing users to specify parent/child relationships between events.

## Background
- RELATIVE-TO is a standard iCalendar property (RFC 5545/5546)
- Defines relationships: PARENT, CHILD, SIBLING
- Used for linking related events (e.g., meeting → follow-up meeting)
- Format: `RELATIVE-TO:UID` with `RELTYPE=PARENT/CHILD/SIBLING`

## Current State
- Event structure in `src/types/index.ts` has no relationship fields
- EventModal shows basic fields: title, dates, recurrence, reminders
- No support for event linking

## Requirements
1. Add `relativeTo` field to event data model
2. Support relationship types: PARENT, CHILD, SIBLING
3. Add UI in EventModal to select related event
4. Show visual indicator of relationship in calendar views
5. Sync with CalDAV (store in RELATED-TO property)

## Data Model Changes

### Update CalendarEvent type
```typescript
// src/types/index.ts
export interface CalendarEvent {
  // ... existing fields
  
  relationships?: {
    relatedTo: string  // UID of related event
    relType: 'PARENT' | 'CHILD' | 'SIBLING'
  }[]
}
```

### CalDAV Sync

Parse from server:
```typescript
// Parse RELATED-TO from iCalendar
const relatedTo = vevent.getFirstPropertyValue('related-to')
const relType = vevent.getFirstPropertyValue('reltype') || 'PARENT'
```

Send to server:
```typescript
// Generate RELATED-TO property
vevent.addPropertyWithValue('related-to', relatedEventUid)
  .setParameter('reltype', relType)
```

## Edge Cases
- Related event deleted - show "[Deleted Event]"
- Circular references - prevent in UI
- Multiple relationships - support array
- Cross-calendar relationships - allow if same account
- Sync conflicts - merge relationship arrays

```typescript
}
```
